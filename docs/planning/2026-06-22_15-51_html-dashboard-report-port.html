◆Architecture Decision: Raw String Builder (No templ)

Approach: Go strings.Builder + JSON Injection

Why not templ? The project has zero templ dependencies today. Adding templ means a build-time code generator, a runtime import (github.com/a-h/templ), and a templ generate step in the build pipeline. The library philosophy favors minimal deps.

How it works: CSS and JS are embedded as Go const strings. Report data is json.Marshal'ed into <script type="application/json"> tags (same security model as samber's @templ.JSONScript). The JS reads data via JSON.parse(document.getElementById(...)). XSS safety comes from JSON injection (never interpolated into HTML) + the JS esc() function for dynamic content + CSP meta tag.

Key insight: This is exactly how samber's template works internally — the data goes in as JSON script tags, not as interpolated template variables. The only difference is templ generates the boilerplate HTML. We replace that with fmt.Fprintf + strings.Builder. Zero new deps.

◆Risks & Mitigations

CSS/JS as Go string constants

Large string constants (CSS ~400 lines, JS ~700 lines) in a single .go file may trigger linter warnings (funlen, gocyclo). Mitigation: Use //nolint:funlen on the const blocks, or split into multiple const declarations across logical sections.

XSS without templ auto-escaping

Raw string builder doesn't have templ's compile-time HTML escaping. Mitigation: Data goes in as JSON script tags (inherently safe — never parsed as HTML). The JS esc() function handles all dynamic HTML generation. CSP meta tag blocks inline event handlers. Fuzz test validates.

Step name collisions in graph

Two steps with identical String() output merge in the graph node map. Mitigation: Already documented as a known limitation. Use StepID as the SVG node ID internally, display Name as the label. Same approach as existing diagram.go's seen map.

Golden file brittleness

Byte-stable golden files break on any template change. Mitigation: Pin ExportedAt to fixed timestamp. Use ReplayEvents with fixed event timestamps. UPDATE_GOLDEN=1 for regeneration. Intentional — catches unintended output changes.
