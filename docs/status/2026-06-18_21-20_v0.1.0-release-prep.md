### 1. CI is 100% RED — GitHub Actions Budget Exhausted

**Every single CI run fails before any job starts.** The error is:

> `The job was not started because an Actions budget is preventing further use.`

This affects ALL 4 jobs (test, lint, coverage, mod-tidy) on EVERY push. The last 5+ runs all show `failure` with 3-6 second duration (too fast to run anything — they fail at startup).

**Root cause:** GitHub Actions billing — the spending limit is set to $0 or the included minutes are exhausted. This is a **billing/admin setting**, not a code or config problem.

**Impact:** We cannot verify CI is green before release. The `v0.1.0` tag was pushed on top of this broken state.

## e) WHAT WE SHOULD IMPROVE! 📈

### Code Quality

1. **`WritePlantUMLString` has 0% coverage** — `WriteGraphvizString` was tested but its PlantUML sibling was not. Asymmetric coverage gap.
2. **158 LSP warnings** (mostly style nits: varnamelen for short variable names, exhaustruct for partial struct initialization, depguard for test imports). These are all in test/example files and are suppressed by `.golangci.yml` for production, but the LSP still surfaces them.
3. **`fromFlowStatus` at 71.4% coverage** — the default case (unknown status string → Pending) is untested.
4. **No fuzz tests** for `ReadEvents` / `LoadReportFromBytes` — these parse external input and would benefit from fuzzing.

### Architecture

5. **`buildReportFromCore` has 7 parameters** — a growing parameter smell. Consider a `ReportCore` struct as the construction input.
6. **`ReplayEvents` cannot reconstruct DAG edges** — events carry no dependency information. Documented as a limitation, but a future "enriched event" format could carry DAG deltas.
7. **`StepRef` uses `Name` as identity** in replay/filter, which collides if two steps share a name. `StepID` helps in reports but events still key on name.

### Process

8. **git-town auto-pushes** — commits appear on GitHub immediately, triggering CI on every commit including WIP. This is surprising behavior for contributors.
9. **No pre-commit hook** for `golangci-lint` — lint issues are only caught by CI (which is currently broken).
10. **No release automation** — the goreleaser config exists but is not wired into a GitHub Actions workflow.
