## e) WHAT WE SHOULD IMPROVE

### Process Problems

1. **Golden file test is fragile and wastes time.** It has been regenerated 6+ times in git history. Every time someone bumps go-output or changes the HTML template, it breaks. The test doesn't catch real bugs — it catches formatting drift. Consider replacing it with a structural validation test (check for presence of expected DOM elements, CSP header, script tags) rather than byte-for-byte comparison.

2. **I didn't commit the golden file immediately.** After regenerating it early in the session, I moved on to other work. When commits happened (by the user or by a previous session), the golden fix was lost. I should have committed it as a standalone fix immediately after verifying it passed.

3. **The v0.7.0 tag has not been created.** The CHANGELOG says `[0.7.0] - 2026-07-22` but there's no git tag. The release is "documented" but not "shipped."

### Code Quality Observations

4. **PlantUML direction is asymmetric.** It only supports 2 of 4 directions (TD and LR). `DirectionUp` and `DirectionLeft` silently fall back to defaults. This is a PlantUML limitation, not a bug, but the API pretends all 4 directions work on all formats.

5. **`DefaultTableColumns` is a mutable package-level `var`.** A consumer could accidentally mutate it (`auditlog.DefaultTableColumns[0] = auditlog.ColumnError`) and affect all subsequent calls. Should be a function returning a fresh slice, or at minimum documented as read-only.

6. **Coverage plateau at 95.6%.** The remaining uncovered lines are all defensive error paths in `WriteMermaidString`/`WriteD2String`/etc. (the `strings.Builder.Write` that never fails). These are genuinely unreachable but keep coverage below 100%.

7. **The `writeGraph` helper is now used by exactly one caller (Graphviz).** Mermaid and PlantUML bypass it. This isn't wrong — each format needs different post-processing — but it means the "shared helper" abstraction has weakened. If Graphviz ever needs post-processing too, `writeGraph` should be removed entirely.

### Documentation Gaps

8. **No README.md update.** The new `WithColumns` and `WithDirection` features are not mentioned in README.md's feature list or examples. Consumers reading the README won't know these features exist.

9. **No website docs update.** The Astro/Starlight documentation site has no mention of the new features.

10. **AGENTS.md test count says "~310" but actual is 319.** Minor but incorrect.
