01What Did You Forget?

doc.go taught deprecated API to every new user

The package doc comment — the first code example anyone sees — used audit.ExportToFile("audit.json") while that method was deprecated. The quick-start example lied about the correct API. Fixed: now uses ExportJSON.

20+ internal references to deprecated methods

After deprecating WriteReportJSON, WriteEventsNDJSON, ExportToFile, ExportEventsToNDJSON, the internal codebase still used them in tests, comments, README, DOMAIN_LANGUAGE.md, FEATURES.md, and STABILITY.md. The deprecation was incomplete — we deprecated the methods but didn't eat our own dog food.

No gosec in CI

CI has govulncheck (dependency CVEs) but not gosec (code-level security analysis). The how-to-golang skill recommends both. govulncheck catches vulnerable dependencies; gosec catches insecure code patterns. Not blocking but should be added.

encoding/json v2 not available

The how-to-golang skill says Go 1.25+ should use encoding/json/v2, but it is not in the Go 1.26 standard library. The JSON v2 proposal has not been accepted into stdlib. This is blocked on upstream Go — documented but cannot be implemented.

02What Is Stupid That We Do Anyway?

Deprecated aliases for an ALPHA library — FIXED

Four deprecated backward-compat aliases (WriteReportJSON, WriteEventsNDJSON, ExportToFile, ExportEventsToNDJSON) existed in an ALPHA library. Nobody has a production dependency on these. They added noise to IDE autocomplete, confused new users, and were never in a released v1.0. Removed.

Two benchmark files with nearly identical names — FIXED

benchmarks_test.go (runtime overhead benchmarks) and benchmark_test.go (export rendering benchmarks + godoc examples). The names differ by one letter. This is a split brain — anyone searching for benchmarks would find both and not know which is which. Consolidated into benchmarks_test.go with clear section headers.

result symlink in working tree — FIXED

The nix build output symlink (result → /nix/store/...) sat in the repo root. It's gitignored but BuildFlow flags it as ERROR. Removed. It will reappear on nix build — this is expected nix behavior.

04Ghost Systems

Deprecated API aliases — ghost system eliminated

The four deprecated aliases were a ghost system: code that exists for an imagined future need (backward compatibility with nonexistent consumers). They added:

  • 4 methods of noise to the Auditor API surface
  • Confusion to IDE autocomplete (8 methods for 4 operations)
  • Maintenance burden — any new Write* method needed a deprecated alias
  • A false signal that this library has real consumers depending on the old names

Status: Eliminated. For an ALPHA library, breaking changes are expected. The deprecated aliases were dead weight.

helpers.go utilities — borderline ghost

CheckNoClobber, HasPointerAddress, and NameCollisions are standalone utilities not integrated into the Export pipeline. They're "here if you need them" functions. CheckNoClobber in particular could be integrated into Export methods as an opt-in via Config, but currently sits as an orphan utility.

Assessment: These are legitimate library utilities. Not ghost systems — they're documented, tested, and provide real value. The integration question (should CheckNoClobber be a Config option?) is a design decision, not a ghost.

05What Could Be Better?

Type model: denormalized status counts

WorkflowReport has 6 manually-denormalized status count fields (SucceededCount, FailedCount, SkippedCount, CanceledCount, PendingCount, RunningCount). The finalizeDenormalized function has a 12-line switch statement to recompute them. Adding a new status requires touching 4+ files.

Verdict: Justified. These fields exist for JSON consumers (Python, JS, shell scripts) that can't call Go methods. The flat fields are the right trade-off for serialization. A StatusCount(status StepStatus) int method for programmatic access would be a nice addition but is not urgent.

Duration uses *float64 (nullable pointer)

DurationMs *float64 uses a nullable pointer for "duration not measured" semantics. This is awkward in Go — callers must check for nil. A dedicated Duration value type with Valid bool + Ms float64 would be more idiomatic, but it changes the JSON schema.

Verdict: Acceptable for now. The pointer pattern is standard Go for nullable JSON fields. Changing it is a breaking schema change for marginal ergonomic benefit.

encoding/json v1 on Go 1.26

The how-to-golang skill recommends encoding/json/v2 for Go 1.25+, but this package is not in the Go 1.26 standard library. The JSON v2 proposal (go-json-experiment) has not been accepted into stdlib. Migration is blocked on upstream Go.

Verdict: Blocked. Document in ROADMAP. No action possible until Go stdlib ships v2.
